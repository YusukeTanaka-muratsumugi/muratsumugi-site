# ムラツムギ公式サイト 運用マニュアル(GitHub編集版)

このサイトの更新は、**①GitHubのウェブ画面で直接編集**するか、**②Claude(Claude Code)に指示**するかの2通りで行います。
どちらもコードの知識はほぼ不要です。

---

## 1. サイトの仕組み(1分で分かる版)

- サイトの**見た目(デザイン)**と**中身(文章・データ)**が分離されています。
- 中身はすべて `content/` フォルダの中の7つのファイルに入っています。

| ファイル | 内容 | 対応ページ |
|---|---|---|
| `content/news.json` | お知らせ記事 | トップ・お知らせ |
| `content/members.json` | メンバー・アドバイザー | メンバー |
| `content/works.json` | 主な実績 | 主な実績 |
| `content/activities.json` | 活動内容 | トップ・活動内容 |
| `content/about.json` | 私たちについて(本文・考え方) | 私たちについて |
| `content/message.json` | 代表メッセージ | 代表メッセージ |
| `content/site.json` | キャッチコピー・noteURL・フォームURL・法人概要 | 全ページ |

- 写真は `assets/uploads/` フォルダに入っています。
- `assets/` のその他(デザイン)や `.html` ファイルは、日常運用では**触りません**。
- GitHub上でファイルを保存(Commit)すると、**1〜2分後にサイトへ自動反映**されます。

---

## 2. どちらの方法を使うか(使い分けの目安)

| やりたいこと | おすすめの方法 |
|---|---|
| 日付・名前・肩書・URLなど**短い文字の修正** | GitHubで直接編集(3章) |
| お知らせの追加、**長い文章の追加・書き換え** | Claudeに指示(4章) |
| **写真**の追加・差し替え | GitHubでアップロード(5章)+Claudeにパス設定を依頼、または全部Claudeに依頼 |
| ページ構成やデザインの変更 | Claudeに指示(4章) |

長い文章をGitHubで直接編集することもできますが、記法ミス(6章)が起きやすいので、慣れるまではClaudeに任せるのが安全です。

---

## 3. GitHubで直接編集する

1. [github.com](https://github.com) にログインし、`muratsumugi-site` リポジトリを開く
2. `content` フォルダ → 編集したいファイル(例: `members.json`)をクリック
3. 右上の**鉛筆アイコン(Edit this file)**をクリック
4. 該当箇所を書き換える
5. 右上の「**Commit changes...**」→ そのまま「**Commit changes**」を押す
6. 1〜2分後、サイトに反映されます

### 編集の例:メンバーの肩書を変える

```json
"name": "田中 佑典",
"role": "代表理事",   ← この「代表理事」の部分だけ書き換える
```

**鉄則:`" "`(ダブルクォート)の内側だけを書き換える。** 記号(`" , { } [ ]`)は消したり増やしたりしない。

### 編集の例:お知らせを1本追加する

`content/news.json` を開くと、記事が `{ ... }` のかたまりで並んでいます。
既存の記事のかたまりを**丸ごとコピーして直後に貼り付け**、末尾のカンマに注意しつつ、中身(id・日付・タイトル・本文)を書き換えるのが安全です。

```json
"items": [
  {
    "id": "news-2026-08-event",
    "date": "2026-08-01",
    "category": "イベント",
    "title": "◯◯を開催します",
    "body": "本文です。\n\n空行(段落分け)は \\n\\n と書きます。"
  },   ← かたまりとかたまりの間にはカンマが必要。最後のかたまりの後ろには不要
  { ...既存の記事... }
]
```

- `id` は他と重複しない半角英数にする
- 本文中の**段落分けは `\n\n`**、改行は `\n` と書く(実際の改行を入れるとエラーになります)

## 4. Claudeに指示する(いちばん確実)

Claude Code(Claude DesktopのCodeタブ)でこのリポジトリを開き、日本語で指示します。
初回は「**このフォルダはNPOのサイトです。まずMANUAL.mdを読んでから作業して**」と伝えると確実です。

指示の例:
- 「お知らせに、9月のワークショップ開催告知を追加して。日時は◯◯、場所は◯◯」
- 「アドバイザーに◯◯大学の△△先生を追加して。写真はこれ(ファイルを渡す)」
- 「主な実績に◯◯の事例を追加して。文章はこれ:…」
- 「代表メッセージの2段落目をこう直して:…」

Claudeが編集からコミット(保存)までやってくれれば、あとは自動でサイトに反映されます。

---

## 5. 写真の追加・差し替え

1. GitHubで `assets/uploads` フォルダを開く
2. 「**Add file**」→「**Upload files**」で写真をアップロード(**ファイル名は半角英数**にする。例: `member-yamada.jpg`)
3. Commitする
4. 使いたい場所のJSONファイルで、`"photo": "assets/uploads/member-yamada.jpg"` のようにパスを設定する

写真は**長辺1600px程度・500KB以下**に縮小してからアップロードすると、サイトが重くなりません。縮小やトリミングが面倒なときは、元データをClaudeに渡して「これを◯◯の写真に設定して」と頼めば、最適化まで含めてやってくれます。

## 6. よくあるトラブル

| 症状 | 対処 |
|---|---|
| 保存したのにサイトが変わらない | 1〜2分待つ→ブラウザを強制再読み込み(Ctrl+Shift+R) |
| 編集した部分のページが**真っ白/表示されない** | JSONの記法エラー(カンマ・クォートの過不足)。GitHubのファイル履歴(History)から直前の版に戻すか、Claudeに「content/◯◯.jsonの記法エラーを直して」と頼む |
| 段落が分かれない | 本文中の `\n\n` が抜けている |
| 写真が表示されない | パスのつづり(大文字小文字も区別)とファイル名が一致しているか確認 |
| 編集ボタン(鉛筆)が出ない | GitHubにログインしているか、リポジトリへの編集権限(下記)があるか確認 |

### メンバーに編集権限を渡すには
リポジトリの「Settings」→「Collaborators」→「Add people」で、メンバーのGitHubアカウントを招待します(無料アカウントでOK)。

## 7. Netlify側の設定(参考)

- サイトはNetlifyで公開されており、GitHubの `main` ブランチが更新されると自動でデプロイされます
- Netlifyの管理画面を日常的に触る必要はありません(独自ドメインやデプロイ状況の確認くらい)
- ※かつて同梱していたブラウザ管理画面(/admin/)は、Netlifyの認証機能の提供終了に伴い廃止しました
